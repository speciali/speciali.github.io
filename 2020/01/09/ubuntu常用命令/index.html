Ubuntu 常用命令

常见问题

  • 修改全局配置文件,并生效
  • 复制文件夹内所有文件到另一个文件夹？

cp -Rf /home/user1/* /root/temp/
将 /home/user1目录下的所有东西拷到/root/temp/下而不拷贝user1目录本身。

文件管理
1.mkdir 创建目录
2.mkdir -p 递归创建目录 例:mkdir A/B/C -p
3.touch 创建文件
4.rmdir 删除空目录
5.rm 删除文件和目录
-i 以进行交互式执行
-f 强制删除
-r 以递归方式删除，可删除递归式目录
6.cd //change directory 改变所在目录
cd ~ 跳转到当前用户的家目
cd - 跳转到上一次所在路径

7.ls // list 查看目录下内容
-a 列举目录中的全部文件，包括隐藏文件
-l 列举目录中细节，包括权限、所有者、群组、大小、创建日期、文件是否是链接等
-h 以人类可读的方式显示文件的大小，如用K、M、G作单位
ls * *表示任意多个字符，也可以没有
?表示一个字符，一定有1个，不能没有
[xn]表示中括号中的任意1个字符
[abcdef]可以写成[a-f]：表示a-f中的任意1个字符
8.tree 以目录树的方式显示文件结构
9.cp // copy 复制文件
-r 递归复制，用于复制整个目录树、子目录以及其他
-f 强制复制
-i 交互式复制
-v 显示拷贝进度

10、mv // move 移动
-f 强制移动
-i 交互式移动
-v 显示移动进度
11. grep // global regular expression print，全局正则表达式打印 在文件中查找某字符

-v              显示不包含匹配文本的所有行（相当于求反）
-n              显示匹配行及行号  例如 grep -n ‘t’ a.txt
-i              忽略大小写

网络操作命令

1、ifconfig // network interfaces configuring 查看和配置网络设备
2、ping // ping 对一个网址发送测试数据包，以此测试网络
3、ssh // secure shell 远程登录Linux
-l 指定用户，
-p 指定端口

例如： 

              1. 无选项参数运行 SSH：ssh 192.168.1.12
              2.指定登陆用户：ssh  -I liu 192.168.1.12    or    ssh liu@192.168.1.12

4、scp // ecure copy 用于在Linux下进行远程拷贝文件的命令

-1  强制scp命令使用协议ssh1  

-2  强制scp命令使用协议ssh2  

-4  强制scp命令只使用IPv4寻址  

-6  强制scp命令只使用IPv6寻址  

-B  使用批处理模式（传输过程中不询问传输口令或短语）  

-C  允许压缩。（将-C标志传递给ssh，从而打开压缩功能）  

-p 保留原文件的修改时间，访问时间和访问权限。  

-q  不显示传输进度条。  

-r  递归复制整个目录。  

-v 详细方式显示输出。scp和ssh(1)会显示出整个过程的调试信息。这些信息用于调试连接，验证和配置问题。   

-c cipher  以cipher将数据传输进行加密，这个选项将直接传递给ssh。   

-F ssh_config  指定一个替代的ssh配置文件，此参数直接传递给ssh。  

-i identity_file  从指定文件中读取传输时使用的密钥文件，此参数直接传递给ssh。    

-l limit  限定用户所能使用的带宽，以Kbit/s为单位。     

-o ssh_option  如果习惯于使用ssh_config(5)中的参数传递方式，   

-P port  注意是大写的P, port是指定数据传输用到的端口号   

-S program  指定加密传输时所使用的程序。此程序必须能够理解ssh(1)的选项

命令格式：  

        scp local_file remote_username@remote_ip:remote_folder  

        或者  

        scp local_file remote_username@remote_ip:remote_file  

        或者  

        scp local_file remote_ip:remote_folder  

        或者  

        scp local_file remote_ip:remote_file                 

        ** 第1,2个指定了用户名，命令执行后需要输入用户密码，第1个仅指定了远程的目录，文件名字不变，第2个指定了文件名  

        ** 第3,4个没有指定用户名，命令执行后需要输入用户名和密码，第3个仅指定了远程的目录，文件名字不变，第4个指定了文件名

系统管理命令

1、stat // status 显示指定文件的详细信息，比ls更详细
2、who // who 显示在线登陆用户
a、last //查看最近用户的登录记录
b、cat /etc/passwd //查看所有用户列表
3、whoami // who am I 显示当前操作用户
4、hostname // host name 显示主机名
5、uname // Unix name 显示系统信息
6、top // top 动态显示当前耗费资源最多进程信息
7、ps // process status 显示瞬间进程状态 ps -aux

-a                    显示终端上的所有的进程，包括其他用户的进程
-u                    显示进程的详细信息
-x                    显示没有控制终端的进程
-w                   显示加宽，以便显示更多的信息
-r                     只显示正在运行的进程

8、kill // kill 杀死进程，可以先用ps 或 top命令查看进程的id，然后再用kill命令杀死进程。kill -9 pid 强制杀进程
9、du // disk usage 硬盘使用率，查看目录大小 du -h /home带有单位显示目录信息
10、df // disk free 查看磁盘大小 df -h 带有单位显示磁盘信息
11、clear // clear 清屏
12、alias // Create your own name for a command 对命令重命名 如：alias showmeit=”ps -aux” ，另外解除使用unaliax showmeit

关机、重启命令

1、reboot // reboot 重启操作系统
2、shutdown // shutdown 正常关机
shutdown -r now 重启系统，shutdown会给别的用户提示
shutdown -h now 立刻关机，其中now相当于时间为0的状态
shutdown -h 20:25 系统在今天的20:25关机
shutdown -h +10 系统再过10分钟关机
3、halt // halt 关机
4、init // initialization
init 0 关机
init 1 重启

Linux软件包管理
1、dpkg (Debian Package)管理工具，软件包名以.deb后缀。这种方法适合系统不能联网的情况下。

比如安装tree命令的安装包，先将tree.deb传到Linux系统中。再使用如下命令安装。

sudo dpkg -i tree_1.5.3-1_i386.deb         安装软件

sudo dpkg -r tree                          卸载软件

2、APT（Advanced Packaging Tool）高级软件工具。这种方法适合系统能够连接互联网的情况。

依然以tree为例

apt-cache search package 搜索包

apt-cache show package 获取包的相关信息，如说明、大小、版本等

apt-cache depends package 了解使用依赖

apt-cache rdepends package 查看该包被哪些包依赖

apt-get source package 下载该包的源代码

sudo apt-get install package 安装包

sudo apt-get install package=version 装指定版本的包

sudo apt-get install package –reinstall 重新安装包

sudo apt-get -f install 修复安装, “-f = –fix-missing”

sudo apt-get remove package 删除包

sudo apt-get remove package –purge 删除包，包括删除配置文件等

sudo apt-get update 更新apt软件源信息

sudo apt-get upgrade 更新已安装的包

sudo apt-get dist-upgrade 升级系统

sudo apt-get dselect-upgrade 使用dselect升级

sudo apt-get build-dep package 安装相关的编译环境

sudo apt-get clean && sudo apt-get autoclean 清理无用的包

sudo apt-get check 检查是否有损坏的依赖

将.rpm文件转为.deb文件

    .rpm为RedHat使用的软件格式。在Ubuntu下不能直接使用，所以需要转换一下。

    sudo alien abc.rpm

注意：

    Ubuntu的软件源文件是/etc/apt/sources.list
    Ubuntu中apt-get的默认安装路径 
    apt-get下载后，软件的所在路径: /var/cache/apt/archives
    Ubuntu默认的PATH为:
    PATH=/home/brightman/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:
    apt-get install安装目录是包的维护者定的，不是用户。 
    系统安装软件一般在/usr/share
    可执行的文件在/usr/bin
    配置文件可能安装到了/etc
    文档一般在/usr/share 
    lib文件在/usr/lib​
